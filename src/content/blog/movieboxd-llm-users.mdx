## Fake Users? How?

Well, I'm glad you asked.

For this, I used an LLM to generate fake reviews. However, not the typical chatbots online, but rather running one locally on my own machine. For this, I used [Ollama](https://ollama.com/).

### Ollama

A quick google search tells us that it's an *"open-source tool designed to run Large Language Models (LLMs) like ***Llama3***, ***Mistral*** and ***Gemma*** locally on your own machine"*

Setting it up was simple:
    1. Run this command in your terminal `curl -fsSL https://ollama.com/install.sh | sh`
    2. Install and run a model: `ollama run gemma4`
    3. CHAT! 😃

For my particular use case, since I was running the models on my laptop, I decided to go with smaller ones such as `gemma4:e2b` and `lfm2.5-thinking`, otherwise, I'd probably fry my machine.


<Callout variant="tip">

I didn't want to keep Ollama running all the time, so I looked up a few commands and found this.
- If you're on Linux, you can stop <b>Ollama</b> with `systemctl stop ollama`. To start it again, first run `systemctl enable ollama`, then `systemctl start ollama`.

</Callout>

So yea, that's how I'm generating fake reviews.

## Architecture

<Architecture></Architecture>

The project is split into **six** packages:
- **<span className="text-yellow-500/90">config</span>** - loads `.env`, `users.json`, `seed_movies.json`
- **<span className="text-purple-400/90">client</span>** - HTTP wrapper that handles login and attaches the JWT to every request
- **<span className="text-blue-400/90">movie</span>** - picks a random movie from the seed list and fetches it from the API
- **<span className="text-orange-400/90">actions</span>** - the actual API calls: `/watched`, `/favorited`, `/watchlisted`, `review`
- **<span className="text-red-400/90">llm</span>** - streams a review from Ollama.
    - Since the official Go package now results in a 404, I used Go's standard libraries `net/http` and `bufio.Scanner` to make requests to the Ollama endpoint
- **<span className="text-green-500/90">seed</span>** - the orchestrator that wires everything together

## The Decision Tree

This is the part I had the most fun with. Each iteration runs through a probability tree to decide what to do:
```
if already watched -> skip

if already watchlisted:
    50% -> mark watched + enter watched branch
    50% -> skip

if fresh:
    50% -> watched branch
    50% -> not watched branch

watched branch:
    60% -> favorite
    50% -> write review
        if review:
            90% -> generate LLM text
            10% -> rating only

not watched branch:
    70% -> watchlist
    30% -> nothing
```

The percentages are made up but they produce realistic looking distributions.

This results in some users having dense watch histories and strong opinions. Some barely touching anything. It ends up feeling like real people using the app.

It is worth mentioning that the loop runs sequentially rather than concurrently.

<Callout variant="cry">
I designed it this way coz my laptop would not survive running multiple Ollama instances in parallel..
</Callout>

## Making it write like a person

The LLM prompt does a lot of work here:

```go
func sentimentFromRating(rating int) string {
	switch rating {
	case 1:
		return "very negative, disappointed"
	case 2:
		return "mostly negative, underwhelmed"
	case 3:
		return "mixed, neither loved nor hated it"
	case 4:
		return "positive, enjoyed it"
	case 5:
		return "very positive, loved it"
	default:
		return "neutral"
	}
}

func buildPrompt(title string, overview string, releaseDate string, personality string, rating int) string {
	return fmt.Sprintf(
		`You are a real movieboxd user with this personality: %s

		You just watched "%s" (%s) and gave it %d out of 5 stars.
		Here is the movie overview: %s

		Write a short, casual movie review (2-4 sentences) that:
		- Reflects your personality
		- Matches the sentiment of a %d/5 rating (%s)
		- Sounds like a real person wrote it, not a critic
		- Does NOT start with "I"
		- Does NOT mention the star rating explicitly

		Only output the review text, nothing else.`,
		personality, title, releaseDate, rating, overview, rating, sentimentFromRating(rating),
	)
}
```

The rating influences the sentiment. As you can see from the code, a **1/5** gets *"very negative, disappointed"* whereas a **5/5** gets a *"very positive, loved it"*. Combining this with the personality, the output actually varies in interesting ways.

One issue was that the model kept leaking its reasoning process into the review text, with literal `<think></think>` blocks appearing in the output. To address this, I wrote a small function using regular expressions to strip out the contents of the `<think></think>` tags.

```go
func stripThinking(s string) string {
    re := regexp.MustCompile(`(?s)<think>.*?</think>`)
    return strings.TrimSpace(re.ReplaceAllString(s, ""))
}
```
